    Semantic Web & Logic Projections

    quicue-kg exports knowledge graph data to RDF, SKOS, and logic programming formats. This page explains the data model, available formats, and when to use each one.

    RDF in 60 seconds

    RDF (Resource Description Framework) stores data as triples: subject-predicate-object statements.

    <kg:ADR-001>  <rdf:type>        <kg:Decision> .
<kg:ADR-001>  <rdfs:label>      "Use PostgreSQL" .
<kg:ADR-001>  <prov:startedAt>  "2026-01-15" .

    Every fact is one triple. Triples compose — you can merge two RDF datasets by concatenating their triples. Subjects and predicates are IRIs (URIs); objects are IRIs or literal values. That's the entire model.

    The advantage: any tool that speaks RDF can query your knowledge graph. SPARQL endpoints, Oxigraph, Jena, rdflib — they all consume this format directly.

    Projection architecture

    All projections live in kg/aggregate/ and follow the same pattern:

    #SomeProjection: {
    index: #KGIndex          // Input: the computed index
    graph: { ... }            // Output: structured data
    summary: { ... }          // Metadata: counts, stats
}

    CUE comprehensions iterate over the index and emit structured output. The projections are one-way: CUE is the source of truth, external formats are interoperability surfaces. You never edit the RDF — you edit the .kg/ CUE files and re-export.

    Format catalog

    RDF serializations

    These three formats express the same RDF data model in different wire formats:

    Format File Type Best for
    N-Triples ntriples.cue #NTriples grep, sort, diff, bulk loading
    Turtle turtle.cue #Turtle Human reading, SPARQL endpoint import
    JSON-LD provenance.cue #Provenance Web APIs, browser consumption

    N-Triples is the simplest: one triple per line, fully expanded IRIs, no prefixes. Perfect for unix pipelines:

    # Export and grep for all provenance activities
cue export .kg/ -e _ntriples.triples --out text | grep 'prov#Activity'

# Count triples by predicate
cue export .kg/ -e _ntriples.triples --out text | awk '{print $2}' | sort | uniq -c | sort -rn

    Turtle adds prefix declarations and groups triples by subject. Produces the same data as N-Triples but readable:

    @prefix kg: <https://quicue.ca/kg#> .
@prefix prov: <http://www.w3.org/ns/prov#> .

kg:ADR-001 a kg:Decision, prov:Activity ;
    rdfs:label "Use PostgreSQL" ;
    prov:startedAtTime "2026-01-15"^^xsd:date .

    cue export .kg/ -e _turtle.document --out text

    JSON-LD is JSON with @context mapping keys to IRIs. The PROV-O, DCAT, and Web Annotation projections all emit JSON-LD:

    cue export .kg/ -e _provenance.graph --out json
cue export .kg/ -e _catalog.dataset --out json
cue export .kg/ -e _annotations.graph --out json

    Semantic vocabularies

    These projections add vocabulary-specific semantics on top of the raw data:

    Projection Vocabulary Type Purpose
    PROV-O W3C Provenance #Provenance Decision audit trails, provenance chains
    DCAT Data Catalog #DatasetEntry Register projects in data catalogs
    Web Annotation W3C Annotation #Annotations Insights/rejected as scholarly annotations
    SKOS Simple Knowledge Org #SKOSTaxonomy Pattern categories as browsable taxonomy

    SKOS is particularly useful for navigating patterns. It maps pattern categories to skos:Concept top concepts and individual patterns to narrower concepts within them:

    # JSON-LD concept scheme
cue export .kg/ -e _taxonomy.graph --out json

# Turtle for import into existing SKOS taxonomies
cue export .kg/ -e _taxonomy.turtle --out text

    Logic programming

    These projections export facts + inference rules, not graph data. They make knowledge computable:

    Projection Runtime Type Terminates?
    Prolog SWI-Prolog #Prolog No (Turing-complete)
    Datalog Soufflé #Datalog Yes (guaranteed)

    Both include 6 inference rules:

    • contributed(Source, Deriv) — transitive provenance (what sources fed into a derivation?)
    • trust(Proto, Level) — trust levels from protocol authority ranks
    • most_authoritative(System, Proto) — best protocol per system
    • shared_pattern(P, Proj1, Proj2) — patterns used across multiple projects
    • active_decision(Id) — accepted decisions
    • actionable(Id) — high-confidence insights
    # Export Prolog program
cue export .kg/ -e _prolog.program --out text > kb.pl

# Query: which patterns are shared between projects?
swipl -l kb.pl -g "shared_pattern(P, A, B), format('~w shared by ~w and ~w~n', [P, A, B]), fail."

# Export Datalog (guaranteed termination, compiles to C++)
cue export .kg/ -e _datalog.program --out text > kb.dl
souffle kb.dl

    When to use Prolog vs Datalog: Use Prolog when you need recursive queries, negation-as-failure, or interactive exploration. Use Datalog when you need guaranteed termination (e.g., automated CI checks) or when evaluating over large datasets (Soufflé compiles to native code).

    Choosing a format

    I want to... Use
    Load into a triplestore (Oxigraph, Jena) Turtle or N-Triples
    Process with unix tools (grep, sort, diff) N-Triples
    Serve from a web API JSON-LD (PROV-O, DCAT, or Annotations)
    Browse patterns as a taxonomy SKOS
    Query provenance chains Prolog or Datalog
    Run automated checks in CI Datalog (always terminates)
    Register in a data catalog DCAT
    Human-readable audit trail Turtle or PROV-O JSON-LD

    Namespace

    All projections use the kg: namespace prefix, which resolves to https://quicue.ca/kg#. The full JSON-LD context is defined in vocab/context.cue and published at kg.quicue.ca/context.jsonld.

    Standard prefixes used across projections:

    Prefix IRI Vocabulary
    kg: https://quicue.ca/kg# quicue-kg types
    prov: http://www.w3.org/ns/prov# W3C Provenance
    dcat: http://www.w3.org/ns/dcat# Data Catalog
    oa: http://www.w3.org/ns/oa# Web Annotation
    skos: http://www.w3.org/2004/02/skos/core# Knowledge Organization
    dcterms: http://purl.org/dc/terms/ Dublin Core Terms
    rdfs: http://www.w3.org/2000/01/rdf-schema# RDF Schema
    rdf: http://www.w3.org/1999/02/22-rdf-syntax-ns# RDF Syntax
